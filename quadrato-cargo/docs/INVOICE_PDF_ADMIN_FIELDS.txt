================================================================================
CUSTOMER INVOICE PDF — ADMIN-EDITABLE FIELDS
================================================================================
Path: Admin → Bookings → [booking] → Invoice (tab)

The customer invoice PDF is ISO A6 (same footprint as the tracking slip). It is
generated by the API after pickup OTP, when "Customer can download invoice" is
enabled.

Company name, address, phone, email, colours, and footer text come from
Admin → Settings (site PDF / branding fields), not from the booking invoice
form.

--------------------------------------------------------------------------------
PER-BOOKING INVOICE FORM (what staff can change)
--------------------------------------------------------------------------------

1) Customer can download invoice
   - Checkbox. If off, the profile PDF download is blocked for that booking.

2) Charges table (one row per line)
   Columns on PDF: # | Item | Wt (kg) | Size (cm) | Line total
   - Item / service: short description (legacy "Shipment Parcel 1 | Parcel 2…"
     text is shortened automatically on the PDF).
   - Wt / Size: optional; prefill from parcel data when empty.
   - Line total: optional amount (numbers as entered; currency is separate).

3) Invoice no. (optional)
   - Shown in the PDF title line after "Invoice ·".

4) Currency
   - Used for line totals, optional breakdown rows, and the final TOTAL bar.

5) Subtotal, Tax, Customs (optional)
   - If filled, each appears as its own row in a small breakdown block on the
     PDF (before the green TOTAL bar). Leave blank to hide that row.

6) Insurance amount (optional)
   - Shown as an "Insurance" row in the same breakdown block (e.g. 500.00).
   - This is separate from "Insurance details" (narrative).

7) Total
   - Shown in the large TOTAL bar. If empty, the PDF tries to use the sum of
     line totals when possible.

8) Insurance details (optional)
   - Free-text block titled "Insurance details" (policy ref, coverage notes).

9) Service description (optional)
   - Free-text block below the charge table (not the old long shipment summary).

10) Notes (optional)
    - Shown as a note at the bottom (customer-facing).

--------------------------------------------------------------------------------
PDF LAYOUT (current behaviour)
--------------------------------------------------------------------------------

- Header: logo or company name, stamp (INVOICE / INT'L INVOICE), and one combined
  line for address · phone · email (when set).
- Barcode + scan ID (and internal ref when it differs).
- Bill from / Bill to: sender and recipient blocks.
- Charge table: no "Items (one row per parcel)" title; no QTY / RATE / TAX
  columns; no separate "Shipment Parcel 1 | …" summary line.
- Optional: Insurance details, Service description.
- Optional: breakdown rows (Subtotal, Tax, Customs, Insurance amount).
- TOTAL bar (currency + total).
- Footer note from site settings.

Discount is not part of the invoice API or PDF anymore.

--------------------------------------------------------------------------------
TECHNICAL NOTES
--------------------------------------------------------------------------------

- Saving the invoice merges into the existing booking invoice document, and
  clears any optional field you empty in the form (so removing tax or insurance
  amount really removes it from the PDF next time).
- Server files: quadrato-cargo/server (and duplicate server/ tree) —
  public.controller.js (PDF HTML), admin.routes.js (PATCH + Zod), booking.model.js
  (public API shape for profile).

--------------------------------------------------------------------------------
TROUBLESHOOTING — “PDF never changes” / old layout (QTY, RATE, Shipment line…)
--------------------------------------------------------------------------------

1) Production (Next.js + separate API)
   - The browser calls same-origin POST `/api/public/bookings/pdf`.
   - Next proxies to Express using server env: API_BASE_URL or BACKEND_API_BASE_URL
     (must point at your live API, not localhost).
   - Redeploy BOTH the frontend and the API after code changes.

2) Local dev
   - From repo root, API is usually: npm run dev:api  →  folder `server/` (port 4010).
   - Restart the Node process after editing public.controller.js so Puppeteer uses new HTML.

3) Verify you are getting the real server PDF
   - Network tab → invoice download response should include header:
     X-QC-Invoice-Template: 2026-04-v2
   - If that header is missing, you are not receiving the Express-generated file.

4) Silent fallback removed for invoices
   - If the API returns 404/401/500, the app no longer swaps in a simplified client PDF
     without telling you; fix the API route or env instead.
